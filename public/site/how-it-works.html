/ HOW IT WORKS

Set it once.
It runs onchain.

Three things you do once, then an autonomous loop that does the rest. No dashboards to babysit, no transactions to sign. Every action is bounded, logged, and provable.

01

Connect in MiniPay

Open RemitRoute inside MiniPay. It detects the wallet and auto-connects. One tap, no seed phrase, no connect button, no signing for recurring actions.

// detected
provider.isMiniPay = true

02

Set a rule in plain language

Type it the way you would say it, or pick a preset. RemitRoute parses it into a typed, capped schedule and reads it back to you before anything moves.

"Save 10 percent every Friday"
↓ parsed
kind: savings_sweep
amount: 10% idle cUSD
cadence: FRI 09:00
03

The agent runs it onchain

Every 20 minutes the agent wakes, checks what is due, and fires the actions. Gas is paid in cUSD, so no one ever needs to hold CELO. Each result is stamped with a proof hash on the live feed.

/ THE HEARTBEAT LOOP

Every 20 minutes, unattended.

A guardian loop runs on a fixed cadence. Health comes first and can block the whole cycle. Money only moves when every guard is green.

GUARD 01
Health check
RPC liveness, fallback ready, gas buffer above the floor. Blocking.
GUARD 02
Load due rules
Every schedule whose next run has arrived, across all users.
GUARD 03
Execute
Enforce caps, lock, build the tx with gas in cUSD, send, reschedule.
GUARD 04
Post metrics
Uptime, success rate, response time, written to ERC-8004 onchain.
GUARD 05
Confirm
Notify the user in-app and on Telegram with the proof hash.
GUARD 06
Safety + halt
Anomaly checks. If volume or failures spike, pause and alert.
